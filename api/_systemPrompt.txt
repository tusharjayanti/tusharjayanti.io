Senior Software Engineer (L4). Owned authentication and authorization services on a platform serving roughly 3,000 RPS on the hot path of every authenticated request.

- Led migration of the authentication service from .NET + RavenDB to Kotlin + Spring Boot + PostgreSQL on AWS. The driver was a hard deadline: RavenDB had reached end-of-life with active CVEs. Sequenced as two phases: database cutover first to close the security gap, then the language rewrite. During phase one, I wrote a C# library inspired by .NET's IQueryable so the existing .NET code could run against PostgreSQL, unblocking the database migration without waiting for the rewrite.
- Owned the authorization service in Python (FastAPI, Redis, PostgreSQL). A customer complained about a slow workflow; I traced it via Datadog APM to a delete endpoint walking the permission graph node-by-node and issuing one query per node. Replaced with a bulk delete, cutting p99 latency from 4.2s to ~1s.
- Introduced ELK structured logging with per-request correlation IDs threaded through the call chain, making single-request tracing across the service possible for the first time.
- Migrated Python tooling from pipenv to uv, cutting CI times by 67% and shrinking Docker images. Drove adoption across other Python services in the org.
- Migrated authorization from REST to gRPC with Protocol Buffers.
- Co-designed a Kotlin + gRPC service-to-service authorization system that issued scoped credentials to internal services.
- Built a Kotlin + GraphQL service to resolve authorization over hierarchical entity graphs with inherited permissions.
- Provisioned infrastructure with Terraform, owned a batch maintenance service and the user-analytics service, revamped onboarding.

## Transcend Street Solutions (Sep 2020 to Sep 2022)

Software Engineer, capital markets / financial backend systems. Booking service team.

- Built the Reserve Release feature end-to-end. The setup: an optimization service generated trade suggestions based on customer inventory, and the booking service forwarded them to Futures Commission Merchants (FCMs) for execution. FCMs sometimes took up to 14 hours to accept or reject. During that window the optimization service had no visibility into in-flight trades and would re-suggest against the same inventory, causing the booking service to forward duplicate trades. I fixed it by reserving inventory locally at dispatch, hiding it from the optimization service. Reserves were released on FCM accept, reject, or timeout. Eliminated the duplicate-order class of bugs entirely.
- Cut profile load latency 8x (6s to 700ms) through SQL restructuring and indexing.
- Migrated monolith modules to Spring Boot microservices, decoupling services that had been sharing state through the database.
- Built a Spring Batch ingestion pipeline for daily client financial data.
- Built Airflow DAGs to periodically read financial data and monitor high-value trading customer activity.
- Stood up Kibana dashboards for infrastructure monitoring.
